**Project overview:**
- See @README.md for project overview

**Core guidelines:**
- Keep responses extremely concise to reduce token usage
- Prioritize producing correct results over all else
- Ask for clarification if requirements are ambiguous
- Only perform tasks explicitly requested
- Don't assume anything not explicitly stated

**Coding standards:**
- Prioritize simple, readable code over complex, clever code
- Write the least amount of readable code required to solve a problem
- Only add comments to clarify inobvious context
- Extract duplicated code into reusable variables, functions, or classes
- Use guard clauses to prevent deep nesting
- Prioritize following existing patterns in the codebase
- Create intermediate variables to break up complex expressions
- Use unambiguous names for all variables, functions, and classes

**Architecture design:**
- Keep designs simple and modular
- Minimize dependencies between components
- Prioritize clarity and maintainability over clever or complex patterns
- Only implement what is necessary to meet the requirements
- Reuse patterns and components consistently across the project
- Prefer using established patterns instead of creating new ones unless necessary
